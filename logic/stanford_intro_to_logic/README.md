## Notes

### Syntax, Semantics, Proof, Models

One compact way to relate the course themes:

* **Syntax**: The formal grammar for writing logical sentences.
* **Semantics**: The meaning of sentences under interpretations or truth assignments.
* **Satisfiability**: Whether there is a model or assignment that makes a sentence true.
* **Proof**: A rule-governed method for deriving conclusions from premises.
* **Induction and equality**: Tools for reasoning about structured objects and identity.

In other words: syntax gives the language; semantics gives meaning; satisfiability checks possible worlds; proof gives derivations; induction and equality extend reasoning to structured domains.

#### Computer science example

* **Propositional logic**: Model digital circuits, constraints, and Boolean programs.
* **Satisfiability**: Determine whether a set of constraints has a solution.
* **Relational logic**: Represent objects, relations, databases, and knowledge bases.
* **Proof systems**: Justify program properties and verify conclusions mechanically.

If a verifier accepts an invalid claim, the failure points back to syntax, semantics, proof rules, or model assumptions.

#### Conceptual chain

You can summarize the course arc like this:

> Syntax → semantics → satisfiability → proof → relational reasoning → induction and equality

The course is about learning to represent claims formally and reason about them with precision.
